# 1\. 概述

本接口规范定义了短信系统 \(服务方\) 与客户系统 \(客户方\) 之间的数据传输协议及使用方法。通过这个接口规范可以实现客户系统 \(通过短信系统\) 与手机终端之间的消息传送。

## 1\.1 名词定义

### 1\.1\.1 下行消息

下行消息又称 MT 消息，是指由业务系统送到短信平台，再转发到运营商的短信网关，并最终发送到手机终端上的文字信息。

### 1\.1\.2 上行消息

上行消息又称 MO 消息，是指由手机终端发送至短信平台并最终推送到客户系统的文字信息。

### 1\.1\.3 状态报告

状态报告是一条特殊的上行消息，用于报告下行消息的投递情况。即消息被送达到手机终端后返回的状态消息。

## 1\.2 工作方式

双方通过 HTTP/HTTPS 协议交互数据，仅支持 `POST` 方式交互。
POST 请求 `Content-Type`：`application/x-www-form-urlencoded`

### GET 方式废弃原因

1. 安全性较差，明文传输数据

2. 传输的数据量较少，无法进行批量群发

3. 中文存在乱码情况，需要进行 URL 编码

> 接口不再支持 GET 方式提交短信。
> 
> 

高并发场景可采用多线程并发提交，提升发送速率。

## 1\.3 字符编码

短信平台收发统一使用 **UTF\-8** 编码，客户业务系统必须同步使用 UTF\-8，否则中文乱码。

URL 转义特殊字符替换规则（半角转全角）：

- 半角 `&` → 全角 `＆`

- 半角 `%` → 全角 `％`

- 半角 `+` → 全角 `＋`

## 1\.4 计费方式

1. **普通短信**：内容≤70 字符，按 1 条计费

2. **长短信**：70＜内容≤1000 字符，每 67 字符计 1 条

### 字符统计规则

汉字、数字、英文、半角 / 全角符号均占 1 字符；emoji 表情占 2 字符。

> 注意：emoji 拆分在两条短信交界处会导致手机乱码。
> 
> 

### 计费包含内容（全部计入字符统计）

- A、短信签名及括号 `【】`

- B、标点、空格、换行、特殊符号

- C、退订提示（如 “退订回复 TD”）

- D、链接、网址

## 1\.5 绑定 IP 要求

接口支持绑定来源 IP，默认最多绑定 2 个 IP；如需更多绑定数量，联系客服处理。

# 2\. 下行消息

## 2\.1 接口地址

`http(s)://xxx.xxx.xxx.xxx:xxxx/HttpSmsMt`

> 实际地址请咨询客服
> 
> 

提交方式：仅 POST
参数格式：key\-value 表单格式 `name=xxx&pwd=xxx`

## 2\.2 提交参数说明

|参数名称|参数说明|备注|
|---|---|---|
|name|帐号 \(必选\)|\-|
|pwd|密码 \(必选\)|32 位小写 MD5；加密规则：`MD5(pwd+mttime)`|
|subid|扩展码 \(可选\)|多签名场景必填；2 位数字；一扩展码对应一个报备签名|
|phone|手机号码 \(必选\)|国内：11 位手机号；国际：00 \+ 国家码 \+ 手机号<br>多号码用英文逗号分隔；单次国内最多 100 个，国际 50 个，可联系客服扩容|
|content|短信内容 \(必选\)|POST 无需 URL 编码；最长 1000 字符；建议将内容中半角`%`替换为全角`％`防乱码|
|mttime|提交时间 \(必选\)|格式`yyyyMMddhhmmss`；必须为当前时间，偏差不可过大；仅用于密码加密，不控制发送时机|
|rpttype|返回数据类型 \(必选\)|0=Xml（过时）；1=Json（推荐）|
|extend|扩展字段|最长 32 位字符串|

## 2\.3 返回值说明

|参数名称|参数说明|备注|
|---|---|---|
|ReqCode|返回码|提交结果编码，详见第 4 章|
|ReqMsg|返回说明|对 ReqCode 的文字描述|
|ReqId|提交批次 ID|用于匹配后续状态报告|

## 2\.4 返回内容示例

### JSON 格式（推荐）

```json
{
    "ReqCode": "00",
    "ReqMsg": "提交成功",
    "ReqId": "16011514475100000001"
}
```

### XML 格式（过时）

```xml
<resp>
    <reqId>16011514475100000001</reqId>
    <reqCode>00</reqCode>
</resp>
```

# 3\. 上行消息 \& 状态报告

客户需提供接收回调 URL，平台主动推送上行短信与状态报告；上行、状态报告可共用同一个回调地址（当前仅支持单 URL）。
状态报告通过 `reqid + phone` 匹配对应下发短信。

## 上行推送示例 URL

```Plain Text
http://xxx.xxx.xxx/xxx?msgtype=0&phone=13400000000&subsubcontent=用户回复内容&receivetime=20160101081220
```

## 状态报告推送示例 URL

```Plain Text
http://xxx.xxx.xxx/xxx?msgtype=2&phone=13400000000&reqid=1601151447510000001&receivetime=20160101081220&sendtime=20160101081218&state=0
```

## 3\.1 上行消息参数说明

|参数名称|参数说明|
|---|---|
|msgtype|0 = 上行消息|
|phone|回复手机号|
|subid|扩展号|
|content|用户上行短信内容|
|receivetime|平台接收上行时间，格式`yyyyMMddHHmmss`|

## 3\.2 短信状态报告参数说明

|参数名称|参数说明|
|---|---|
|msgtype|2 = 状态报告|
|phone|接收短信的手机号|
|reqid|下发接口返回的提交批次 ID ReqId|
|receivetime|平台收到状态报告时间，`yyyyMMddHHmmss`|
|sendtime|短信下发时间，`yyyyMMddHHmmss`|
|state|投递状态码（详见第 5 章）|
|extend|下发时传入的扩展字段|

## 3\.3 返回值

客户服务端收到平台推送后，必须返回纯文本 `ok` 表示接收成功。

# 4\. 提交返回值（下发接口 ReqCode）

|ReqCode|返回说明|备注|
|---|---|---|
|00|提交成功 / 查询成功|\-|
|01|提交参数异常|缺少必选参数、参数格式错误|
|02|手机号参数异常|号码格式非法|
|03|扩展号参数异常|subid 不合规|
|04|提交时间参数异常|mttime 时间偏差过大或格式错误|
|05|短信内容解析异常|content 编码 / 特殊字符解析失败|
|06|提交内容超长|content 超过 1000 字符上限|
|07|扩展参数超长|extend 最大 32 位字符串|
|10|IP 认证失败|调用 IP 未绑定白名单|
|11|帐号密码认证失败|name/pwd/md5 加密错误|
|12|余额查询失败|后付费客户无需关注|
|19|余额查询超速|每分钟最多查询 1 次|
|99|系统内部错误|平台服务异常|
